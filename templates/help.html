TimeGen: User Manual

Automated Academic Timetable Generation System

1. Data Integrity & Naming Rules

The system is strict about spelling but forgiving with formatting. Consistency across both files is mandatory.

  • Case & Spaces: Discrete Maths and discrete maths are treated as the same.
  • Literal Spelling: Discrete Maths and DiscreteMaths are treated as different courses.
  • Class Specificity: "Maths" in class S1 is scheduled independently of "Maths" in class S3.
  • Faculty Names: Prof. Smith and Prof Smith (missing period) are treated as two different people.

2. Course-Teacher Mapping

This file defines your department's workload. Only include faculty from your department here.

Course Types (L vs O)

  • Type L (Lecture): Standard distribution. Workload is split among listed teachers.
  • Type O (Simultaneous): All listed teachers are locked into the same period at once.

Example: Parallel Physics and C Labs

To run two different labs simultaneously for the same class, treat them as one entry:

ClassCourseTypeHoursTeacher
S1APhysics / C LabO3Prof. A, Prof. B, Prof. C

Example: Lecture + Tutorial Split

If a course has 2 lecture hours (1 teacher) and 1 tutorial hour (multiple teachers):

ClassCourseTypeHoursTeacher
S1AMathsL2Prof. Smith
S1AMaths (T)O1Prof. Smith, Prof. Davis

3. Partially Filled Timetable

Use this to manually "lock" periods before the algorithm starts.

  • Breaks: Use BREAK or LUNCH to block slots for all classes.
  • Internal Subjects: Enter the name exactly as it appears in the Mapping file to allocate your faculty.
  • External Subjects: If a class is taken by another department (e.g., Physics), enter it here. Since there is no matching teacher in your mapping file, the system will block the students' time without affecting your faculty workload.
  • Remedial/Free: Label slots as REMEDIAL to reserve time without assigning a teacher.

4. Advanced Features

Unsuitable Slots

Used to block faculty who are busy with administrative duties or taking classes in other departments. Access this via the "Block Teacher Availability" menu after uploading your mapping file.

Intentional Double-Booking

For combined seminars or shared electives where one teacher handles two classes at once, place the same subject in the same slot for both classes in the Partial file. The system will merge these in the output (e.g., "S1A, S1B").

5. Alerts & Warnings

[SOFT WARNING] Teacher Continuity Forced: System assigned back-to-back periods to fulfill hours.
[STRONG WARNING] Double Booking: Detected a manual double-assignment in your Partial file.
[HARD ERROR] Failed to Allocate Period: Constraints were mathematically impossible. You must manually assign this period in the exported Excel file.
