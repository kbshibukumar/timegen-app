TimeGen: The Complete Reference Guide

Based on ICCC 2025 Paper: "A Novel Slot Filling Algorithm for Automated Time Table Generation"

1. Data Integrity & Naming Rules

The algorithm uses "Fuzzy-insensitive but Literal" matching. Consistency across your Excel files is mandatory for the logic to work.

  • Case & Spaces: The system is case-insensitive. Discrete Maths and discrete maths are seen as identical.
  • Literal Spelling: Spelling must be exact. Discrete Maths and DiscreteMaths (missing space) are treated as different courses.
  • Abbreviations: The system cannot guess meanings. Maths and Mathematics are considered separate entities.
  • Class-Specific Logic: "Maths" in class S1 is scheduled independently of "Maths" in class S3.
  • Faculty Names: Prof. Smith and Prof Smith (missing period) or Prof.Smith (missing space) are treated as three different people. Always use one exact format.

2. Course-Teacher Mapping

This file defines your department's workload. Only include faculty from your department here, as this file calculates your internal departmental workload.

Course Types (L vs O)

  • Type L (Lecture): Standard distribution. The workload hours are split among the listed teachers.
  • Type O (Other / Simultaneous / Parallel): All listed teachers are locked into the same slot simultaneously.

Example: Parallel Physics and C Labs

If half the class is in Physics Lab while the other half is in C Lab simultaneously, treat them as a single entry:

ClassCourseTypeHoursTeacher
S1APhysics / C LabO3Prof. A, Prof. B, Prof. C

Example: Mixed Lecture & Tutorial Split

If a course has 2 lecture hours (1 teacher) and 1 tutorial hour (multiple teachers):

ClassCourseTypeHoursTeacher
S1AMathsL2Prof. Smith
S1AMaths (T)O1Prof. Smith, Prof. Davis

3. Partially Filled Timetable

Used to "hard-lock" fixed periods before the automated algorithm begins.

  • Breaks: Use BREAK or LUNCH to block slots globally for all classes.
  • Internal Subjects: Enter the course name exactly as it appears in the Mapping file to auto-allocate your departmental faculty.
  • External Department Subjects: If a class is handled by another department (e.g., Physics), enter it here. Since there is no matching teacher in your departmental mapping file, the system will block the students' time but will not assign any of your faculty to that slot.
  • Remedial / Free Periods: Label slots as REMEDIAL to reserve time for students without allocating a teacher.

4. Advanced Constraints

Unsuitable Slots (Teacher Blocking)

Use this for faculty who are unavailable due to administrative duties, research, or taking classes in other departments. Access this via the "Block Teacher Availability" menu after uploading your mapping file.

Intentional Double-Booking (Combined Classes)

For shared electives or combined seminars where one teacher handles two classes at once (e.g., S1A and S1B in a main hall):

  1. In the Partial Timetable, place the same subject in the same slot for both classes.
  2. In the Mapping File, ensure the same teacher is assigned to that course name for both classes.

The system will flag this with a warning but allow the booking, merging the faculty workload entry as "S1A, S1B".

5. Alerts & Warnings Decoder

[SOFT WARNING] Teacher Continuity Forced: The system assigned back-to-back periods to a teacher to ensure all required syllabus hours were met.
[STRONG WARNING] Double Booking in Partial File: Detected that you manually scheduled a teacher for two classes at once. Verify if this was an intentional combined session.
[HARD ERROR] Failed to Allocate Period: Constraints were mathematically impossible (e.g., the teacher was blocked or busy during all available hours for that class). Action: You must manually assign this missing period in the exported Excel file.
